# 🏃 Sprint 1: MVP 版本（核心功能）

**时间**: 2026-03-23 ~ 2026-04-05 (2 周)  
**主题**: "从 0 到 1 - 让想法变成代码"  
**目标**: 实现最小可行产品，用户能够输入产品想法，AI 自动生成 PRD、用户画像、竞品分析，并创建基础代码框架

## 🎯 Sprint 1 目标

### 核心价值主张
- ✅ 用户能够通过自然语言描述产品想法
- ✅ AI 自动生成完整的产品文档（PRD + 用户画像 + 竞品分析）
- ✅ AI 自主创建基础代码框架
- ✅ 可视化的开发进度追踪
- ✅ 基础的 Git 版本控制集成

### 成功标准
- [ ] 端到端流程可演示（Idea → PRD → Code）
- [ ] 核心功能可用性 > 90%
- [ ] 关键路径无阻塞性 Bug
- [ ] 至少生成 1 个真实可运行的示例项目

## 📦 Sprint 1 功能范围

### 1. Vibe Design - 产品设计模块（100% 完成）

**核心功能**:
- ✅ **PRD 生成** (`VD-001 ~ VD-005`)
  - 用户输入产品想法
  - AI 生成结构化 PRD 文档
  - 支持 MiniMax API 真实调用
  - 基本 UI 展示和编辑功能

- ✅ **用户画像生成** (`VD-006`)
  - AI 生成 3-5 个典型用户画像
  - 包含基本信息、目标、痛点
  - 可视化卡片展示

- ✅ **竞品分析** (`VD-007`)
  - AI 识别主要竞争对手
  - 生成对比分析表格
  - SWOT 分析基础框架

- ✅ **流程整合** (`VD-008 ~ VD-012`)
  - 顺序工作流：想法 → PRD → 画像 → 竞品
  - 状态管理和导航
  - 基础数据持久化

**验收标准**:
- [ ] 用户能够在 5 分钟内完成产品构思
- [ ] 生成的 PRD 包含完整的功能列表和用户故事
- [ ] 用户画像具有代表性和差异化
- [ ] 竞品分析包含至少 3 个竞争对手

### 2. Vibe Coding - AI 编码模块（核心子集）

**核心功能**:
- ✅ **Initializer Agent** (`VC-001, VC-002`)
  - PRD 解析为基础任务列表
  - 环境检查（Git/Node.js/Rust）
  - 项目脚手架创建
  - 基础 Git 初始化

- ✅ **Coding Agent 基础** (`VC-012 ~ VC-014`)
  - 单个 Agent 执行逻辑
  - 简单的并发控制（2+ Agents）
  - 基础分支管理

- ✅ **MR Creation Agent** (`VC-016`)
  - 代码合并功能
  - 基础冲突检测
  - AI 生成 MR 描述

- ✅ **质量门禁基础** (`VC-019, VC-020, VC-026`)
  - 代码生成提示词模板
  - 文件应用器（写文件）
  - Git 提交助手（Conventional Commits）

- ✅ **UI 组件** (`VC-023 ~ VC-025`)
  - 文件浏览器（基础树形结构）
  - 代码编辑器（语法高亮）
  - 终端模拟器（基础输出显示）

**验收标准**:
- [ ] Initializer Agent 能根据 PRD 创建至少 10 个开发任务
- [ ] Coding Agent 能独立完成简单功能的代码实现
- [ ] 生成的代码能通过基础 lint 检查
- [ ] Git 提交记录规范清晰

### 3. Vibe Marketing - 营销模块（基础版）

**核心功能**:
- ✅ **发布策略基础** (`VM-001 ~ VM-003`)
  - 基础发布计划生成
  - 简单的时间线展示
  - 关键里程碑标记

- ✅ **营销文案基础** (`VM-004, VM-005`)
  - AI 生成产品描述文案
  - 社交媒体文案草稿
  - 基础渠道适配（Twitter/公众号）

**验收标准**:
- [ ] 生成包含 3-5 个关键日期的发布计划
- [ ] 提供至少 5 条可用的营销文案
- [ ] 文案风格符合产品定位

### 4. 基础设施（核心支撑）

**核心功能**:
- ✅ **Tauri + React + Rust 架构** (`INFRA-001 ~ INFRA-005`)
  - 项目基础框架
  - 跨平台构建能力
  - 前后端通信机制

- ✅ **数据库基础** (`INFRA-006 ~ INFRA-010`)
  - SQLite 集成
  - 基础表结构设计
  - CRUD 操作封装

- ✅ **AI 适配器（MiniMax）** (`INFRA-011 ~ INFRA-014`)
  - MiniMax API 完整接入
  - 5 个核心 API 方法
  - 错误处理和重试机制

- ✅ **工具检测** (`INFRA-015, INFRA-016`)
  - Git 安装检测
  - Node.js/Rust 环境检查

**验收标准**:
- [ ] 应用能在 Windows/macOS/Linux 上运行
- [ ] 数据库能正确存储和检索项目数据
- [ ] MiniMax API 调用成功率 > 95%

## 🚧 Sprint 1 已知限制

### 功能限制（留到 Sprint 2）
- ❌ 不支持多 AI 厂商切换（仅 MiniMax）
- ❌ 不支持流式输出（打字机效果）
- ❌ 没有实时审查和性能监控
- ❌ 没有 WebSocket 实时通信
- ❌ 没有高级质量门禁（ESLint/单元测试自动运行）
- ❌ 没有 HITL 检查点（人工审查流程）
- ❌ 没有复杂的依赖管理和任务调度

### 性能限制
- ⚠️ 响应时间可能 > 5 秒（无优化）
- ⚠️ 并发 Agent 数量限制为 2-4 个
- ⚠️ 没有缓存和复用机制
- ⚠️ 内存使用未优化

```
基础数据库          ────────────→ 完整数据模型

基础 UI            ────────────→ 高级可视化

```

### 平滑过渡策略

1. **向后兼容**: Sprint 2 的新功能必须兼容 Sprint 1 的数据格式
2. **渐进式升级**: 用户可以继续使用 Sprint 1 的功能，同时选择是否启用 Sprint 2 的新特性
3. **数据迁移**: 提供自动化脚本将 Sprint 1 的项目数据迁移到 Sprint 2 的增强模型
4. **功能开关**: 通过配置开关控制 Sprint 2 功能的启用，避免影响现有用户
