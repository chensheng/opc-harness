## Context

README.md 是项目的门面文档，应该简洁明了，让新用户在 30 秒内理解项目价值，提供快速上手的必要信息，并引导用户到详细文档获取更多信息。

当前 README.md 共 178 行，包含大量技术细节和工作流说明，这些信息更适合放在专门的文档中（如 AGENTS.md、OpenSpec specs）。

## Goals / Non-Goals

**Goals:**
- 将 README.md 精简至约 65 行（减少约 63%）
- 保留概述、核心功能、快速开始三个核心部分
- 添加简洁的文档导航链接
- 保持文档的专业性和完整性

**Non-Goals:**
- 不修改核心功能描述
- 不改变快速开始的安装和运行步骤
- 不涉及代码变更
- 不删除重要的项目信息

## Decisions

### Decision 1: 保留的三个核心部分

**选择**: 保留概述、核心功能、快速开始

**理由**:
- 概述：让用户快速理解项目是什么
- 核心功能：展示项目的三大模块和价值
- 快速开始：提供立即上手的必要步骤

这三个部分是 README 的标准结构，符合开源项目的最佳实践。

### Decision 2: 删除的部分及替代方案

**删除内容**:
1. 开发工作流（第 66-138 行，73 行）
2. 技术架构（第 139-160 行，22 行）
3. Harness Engineering（第 161-178 行，18 行）

**替代方案**:
- 在 README 末尾添加"了解更多"章节，链接到：
  - `[AGENTS.md](./AGENTS.md)` - 完整的开发工作流、技术架构和 Harness Engineering 说明
  - `openspec/specs/` - OpenSpec 能力规范文档
  - `openspec/changes/archive/` - 历史变更记录

**理由**:
- 避免信息重复
- 保持 README 简洁
- 引导用户到专门的文档获取详细信息

### Decision 3: 新增"了解更多"章节

**内容**:
```markdown
## 📚 了解更多

- **[AGENTS.md](./AGENTS.md)** - 完整的开发工作流、技术架构和 AI Agent 导航 ⭐
- **openspec/specs/** - OpenSpec 能力规范文档
- **openspec/changes/archive/** - 历史变更记录
```

**位置**: 在快速开始章节之后

**理由**:
- 为用户提供进一步探索的路径
- 突出 AGENTS.md 的重要性
- 保持文档的可发现性

## Risks / Trade-offs

**风险**:
- 用户可能找不到某些技术细节 → **缓解**: 通过"了解更多"链接引导
- 现有用户习惯当前 README 结构 → **缓解**: 核心信息保留，只是重组

**权衡**:
- 简洁性 vs 完整性 → 选择简洁性，完整性通过链接保证
- 自包含 vs 模块化 → 选择模块化，符合现代文档最佳实践
