## Context

AGENTS.md 是项目的 AI Agent 导航地图，当前共 559 行，包含大量 OpenSpec 工作流详细说明、最佳实践、常见问题等内容。这导致 AI Agent 和开发者难以快速找到关键信息，且与 OpenSpec specs 中的内容重复。

## Goals / Non-Goals

**Goals:**
- 将 AGENTS.md 从 559 行精简至约 150-200 行（减少约 65-73%）
- 保留快速入口章节（按优先级列出的核心文档链接）
- 保留三大支柱概述（上下文工程、架构约束、反馈回路）
- 删除或简化冗长的教程和最佳实践说明
- 添加简洁的文档导航链接

**Non-Goals:**
- 不修改核心功能描述
- 不改变快速入口的文档链接
- 不涉及代码变更
- 不删除重要的导航信息

## Decisions

### Decision 1: 保留的核心章节

**选择**: 保留快速入口、三大支柱概述、前端 Console Bridge（如果存在）

**理由**:
- 快速入口：让 AI Agent 和开发者快速找到最重要的文档
- 三大支柱概述：提供项目核心理念的高层次理解
- 这些部分是 AGENTS.md 的核心价值，符合"导航地图"的定位

### Decision 2: 删除或简化的部分

**删除/简化内容**:
1. OpenSpec 快速入门详细教程（约 100+ 行）→ 简化为简洁引用
2. OpenSpec 最佳实践长篇说明（约 150+ 行）→ 移至 development-workflow spec
3. 常见问题 FAQ 章节（约 100+ 行）→ 简化为链接或删除
4. 三大支柱详细描述（约 100+ 行）→ 保留要点，删除冗余解释

**替代方案**:
- 在 AGENTS.md 末尾添加"详细文档"章节，链接到：
  - `openspec/specs/development-workflow/spec.md` - OpenSpec 工作流详细说明
  - `openspec/specs/agent-initialization/spec.md` - Agent 初始化规范
  - 其他相关的 OpenSpec specs

**理由**:
- 避免信息重复
- 保持 AGENTS.md 简洁，专注于导航
- 引导用户到专门的文档获取详细信息

### Decision 3: 精简策略

**方法**: 
- 采用"导航优先"原则 - AGENTS.md 应该是指南针，不是百科全书
- 每个章节控制在 20-30 行以内
- 使用链接而非长篇说明
- 保留代码示例但简化解释

**理由**:
- 符合现代文档最佳实践
- 提升 AI Agent 的阅读效率
- 降低维护成本

## Risks / Trade-offs

**风险**:
- 用户可能找不到某些详细说明 → **缓解**: 通过"详细文档"链接引导
- 现有用户习惯当前 AGENTS.md 结构 → **缓解**: 核心导航保留，只是重组

**权衡**:
- 简洁性 vs 完整性 → 选择简洁性，完整性通过链接保证
- 自包含 vs 模块化 → 选择模块化，符合现代文档最佳实践
