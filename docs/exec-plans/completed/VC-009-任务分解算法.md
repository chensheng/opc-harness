## 📋 任务概述

### 背景
在 VC-006 和 VC-007 的基础上，Initializer Agent 已经能够解析 PRD 文档和检查环境。现在需要将 PRD 解析结果转化为可执行的开发任务（Issues），这是连接产品设计和编码实现的桥梁。

### 目标
从 MVP 规划中拆解的核心目标：
- **业务目标**: 将产品需求自动转化为可执行的开发任务，减少人工分解成本
- **功能目标**: 基于 AI 实现智能任务分解，生成包含优先级、工时估算、依赖关系的 Issues
- **技术目标**: 优化现有任务分解算法，提升分解质量和准确性

### 范围
明确包含和不包含的内容：
- ✅ **In Scope**: 
  - 完善任务分解提示词模板
  - 优化 Issue 数据结构映射
  - 实现依赖关系识别
  - 添加工时估算逻辑
  - 增加优先级映射规则
  - 编写单元测试覆盖
- ❌ **Out of Scope**: 
  - Issue 的自动创建和推送（VC-010）
  - 任务分配算法（VC-011）

## 💡 解决方案设计

### 架构设计
```
PRD 文档 → PRDParser → 任务分解提示词 → AI 服务 → Issues JSON → 数据映射 → TaskDecompositionResult
                                              ↓
                                        Priority/Status/Dependencies
```

### 核心接口/API
现有接口（已优化）：
```rust
// PRDParser::decompose_tasks()
pub async fn decompose_tasks(
    &self,
    product_name: &str,
    product_description: &str,
    core_features: &[String],
    tech_stack: &[String],
) -> Result<Vec<Issue>, String>

// InitializerAgent::decompose_tasks()
pub async fn decompose_tasks(
    &self,
    prd_result: &PRDParseResult,
) -> Result<TaskDecompositionResult, String>
```

### 数据结构
已优化的数据结构：
```rust
pub struct TaskDecompositionResult {
    pub success: bool,
    pub issues: Vec<Issue>,
    pub total_estimated_hours: f32,
    pub suggested_order: Vec<String>,
    pub dependencies: Vec<(String, String)>,
    pub risks: Vec<String>,
}

pub struct Issue {
    pub issue_id: String,
    pub title: String,
    pub description: String,
    pub priority: Priority,
    pub status: IssueStatus,
    pub estimated_hours: Option<f32>,
    // ... 其他字段
}
```
