## 🎯 技术方案

### 技术栈选择

| 层级 | 技术 | 理由 |
|------|------|------|
| HTTP 层 | Native Fetch API | 原生支持，零依赖，快速 |
| 组件层 | React Testing Library | 已有依赖，模拟用户交互 |
| ~~浏览器层~~ | ~~Puppeteer Core~~ | ~~因网络问题放弃~~ |

### 方案对比

#### 原方案（Playwright）
```
❌ 需要下载浏览器 (~300MB)
❌ 网络问题导致无法安装
❌ 学习曲线陡峭
❌ 独立于现有测试框架
❌ 执行速度慢（10-30 秒/测试）
```

#### 新方案（Vitest + RTL + Fetch）
```
✅ 零额外重依赖
✅ 复用现有技术栈
✅ 学习成本低
✅ 完全集成到 Vitest
✅ 执行速度快（1-3 秒/测试）
✅ 轻量易维护
```
