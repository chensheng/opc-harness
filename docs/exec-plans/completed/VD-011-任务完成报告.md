#### 核心实现

**Kimi Provider 配置**
```rust
impl AIProvider {
    fn get_base_url(&self) -> String {
        match self.provider_type {
            AIProviderType::Kimi => "https://api.moonshot.cn/v1".to_string(),
            // ... 其他 provider
        }
    }

    fn get_auth_header(&self) -> (String, String) {
        match self.provider_type {
            AIProviderType::Anthropic => ("x-api-key".to_string(), self.api_key.clone()),
            _ => ("Authorization".to_string(), format!("Bearer {}", self.api_key)),
        }
    }
}
```

**Kimi 聊天方法**
```rust
async fn chat_kimi(&self, request: ChatRequest) -> Result<ChatResponse, AIError> {
    // Kimi uses OpenAI-compatible API
    self.chat_openai(request).await
}

async fn stream_chat_kimi<F>(
    &self,
    request: ChatRequest,
    on_chunk: F,
) -> Result<String, AIError>
where
    F: FnMut(String) -> Result<(), AIError>,
{
    // Kimi uses OpenAI-compatible API
    self.stream_chat_openai(request, on_chunk).await
}
```

#### 技术特点

1. **OpenAI 兼容 API**: Kimi 使用与 OpenAI 完全兼容的 API 格式
   - Base URL: `https://api.moonshot.cn/v1`
   - 认证方式：`Authorization: Bearer {api_key}`
   - 请求格式：与 OpenAI 相同

2. **支持的模型**
   - `kimi-k2` - Kimi 最新模型
   - `kimi-k2-0711` - Kimi K2 版本
   - `moonshot-v1-8k` - 8K 上下文窗口
   - `moonshot-v1-32k` - 32K 上下文窗口
   - `moonshot-v1-128k` - 128K 超大上下文窗口

3. **复用 OpenAI 实现**: 
   - [chat_kimi()](file://d:\workspace\opc-harness\src-tauri\src\ai\mod.rs#L426-L429) 直接调用 [chat_openai()](file://d:\workspace\opc-harness\src-tauri\src\ai\mod.rs#L169-L225)
   - `stream_chat_kimi()` 直接调用 `stream_chat_openai()`
   - 减少代码重复，易于维护

### 关键设计决策

#### 1. 为什么 Kimi 复用 OpenAI 实现？

```rust
async fn chat_kimi(&self, request: ChatRequest) -> Result<ChatResponse, AIError> {
    // Kimi uses OpenAI-compatible API
    self.chat_openai(request).await
}
```

**原因**:
- ✅ Kimi API 完全兼容 OpenAI 格式
- ✅ 相同的请求/响应结构
- ✅ 相同的认证方式 (Bearer Token)
- ✅ 减少代码重复，降低维护成本
- ✅ 符合 DRY (Don't Repeat Yourself) 原则

#### 2. Kimi vs 其他厂商的 API 对比

| 厂商 | Base URL | 认证方式 | 兼容性 |
|------|----------|----------|--------|
| **OpenAI** | `https://api.openai.com/v1` | `Authorization: Bearer` | 原生 |
| **Kimi** | `https://api.moonshot.cn/v1` | `Authorization: Bearer` | ✅ 完全兼容 OpenAI |
| **GLM** | `https://open.bigmodel.cn/api/paas/v4` | `Authorization: Bearer` | ✅ 兼容 OpenAI |
| **Anthropic** | `https://api.anthropic.com/v1` | `x-api-key` | ❌ 独立格式 |
| **MiniMax** | `https://api.minimax.chat/v1` | `Authorization: Bearer` | ✅ 兼容 OpenAI |

#### 3. Kimi 模型选择策略

```rust
// Kimi 支持的模型列表
let models = vec![
    "kimi-k2",           // 最新模型，性能最优
    "kimi-k2-0711",      // K2 特定版本
    "moonshot-v1-8k",    // 8K 上下文，适合短文本
    "moonshot-v1-32k",   // 32K 上下文，适合中等长度
    "moonshot-v1-128k",  // 128K 超大上下文，适合长文档分析
];
```

**选择建议**:
- **短文本对话**: `moonshot-v1-8k` - 快速、经济
- **PRD 生成**: `moonshot-v1-32k` - 平衡性能和成本
- **长文档分析**: `moonshot-v1-128k` - 处理超长上下文
- **高质量输出**: `kimi-k2` - 最新最强模型
