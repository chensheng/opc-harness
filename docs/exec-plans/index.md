# 执行计划文档说明

> 本目录存储所有活跃和已完成的执行计划

## 📂 目录结构

```
exec-plans/
├── index.md              # 当前文件
├── active/               # 活跃的执行计划
├── completed/            # 已完成的执行计划
└── tech-debt-tracker.md  # 技术债务追踪
```

## 📋 使用说明

- 所有未来完成的任务报告都应直接放在 `completed` 目录
- 活跃的执行计划文档应放在 `active` 目录
- 查看 [MVP 版本规划](../product-specs/mvp-roadmap.md) 了解任务优先级
- 参考 [Harness Engineering 流程](../HARNESS_ENGINEERING.md) 执行开发任务
