# 技术债务追踪指引

## 新境技术债务流程

1. **发现问题**: TODO/FIXME、Code Review、性能瓶颈
2. **评估优先级**: 使用决策矩阵（见下方）
3. **创建文档**: `docs/exec-plans/tech-debts/TD-XXX-description.md`
4. **填写内容**: 基本信息、问题描述、解决方案、验收标准

---

## 优先级决策矩阵

```
                影响范围
              大        小
          ┌─────────┬─────────┐
        高│ P1      │ P2      │
紧急度   │ (高优先)│ (中优先)│
          ├─────────┼─────────┤
        低│ P0      │ P3      │
          │(紧急)  │ (低优先)│
          └─────────┴─────────┘
```

**示例**:
- P0: 编译错误、安全漏洞
- P1: 功能缺陷、性能问题
- P2: 代码异味、测试不足
- P3: 文档缺失、工具优化

---

## 文档规范

### 命名格式

```
TD-XXX-简短描述.md
```

**示例**: `TD-001-rust-option-conversion.md`

### 文档模板

```
# TD-XXX: [中文标题]

## 📋 基本信息
- **创建日期**: YYYY-MM-DD
- **优先级**: P0-P3
- **状态**: 📋 待解决 / 🔄 进行中 / ✅ 已解决
- **影响范围**: [文件/模块]

## 📝 问题描述
[详细说明]

## 🎯 解决方案
[步骤和代码示例]

## ✅ 验收标准
- [ ] [验收项 1]
- [ ] [验收项 2]
```

### 状态流转

```
📋 待解决 → 🔄 进行中 → ✅ 已解决
```
