# 执行计划创建指引

**描述**: 必须严格按照此指引创建执行计划，以便后续的任务跟踪和评估。

**步骤**:

## 1. 创建文件

在 `docs/exec-plans/active/` 目录创建新文件：

```bash
# 文件名：{TASK_ID}-{TASK_NAME}.md
# 示例：US-019-测试任务.md
```

## 2. 填写基本信息

```
# {TASK_ID}: {TASK_NAME}

**状态**: 🔄 进行中
**优先级**: P0/P1/P2
**任务类型**: User Story/Techical Task/Bug Fix/Tech Debt
**开始日期**: YYYY-MM-DD
**预计完成**: YYYY-MM-DD
**负责人**: OPC-HARNESS Team
**关联需求**: _关联的 Sprint 任务 ID_
```

## 3. 填写任务概述

```
## 📋 任务概述

### 背景
_为什么需要这个功能？解决什么问题？_

### 目标
从 Sprint 规划中拆解的核心目标：
- [ ] **业务目标**: _用户价值_
- [ ] **功能目标**: _功能范围_
- [ ] **技术目标**: _技术难点/架构改进_

### 范围
明确包含和不包含的内容：
- ✅ **In Scope**: _包含的功能模块_
- ❌ **Out of Scope**: _不包含的内容_

### 关键结果 (Key Results)
可量化的成功标准：
- [ ] KR1: _例如：Health Score 100/100_
- [ ] KR2: _例如：测试覆盖率 >95%_
- [ ] KR3: _例如：E2E 测试通过率 100%_
```

## 4. 设计解决方案

```
## 💡 解决方案设计

### 架构设计
_系统架构图、组件关系、数据流等_

### 核心接口/API
_主要的函数签名、接口定义_

### 数据结构
_关键的数据模型、类型定义_

### 技术选型
_使用的技术栈、框架、库及选型理由_
```

## 5. 保存并确认

- ✅ 文件位于 `active/` 目录
- ✅ 文件名：`{TASK_ID}-{TASK_NAME}.md`
- ✅ 所有占位符已替换
- ✅ 任务概述清晰完整
