## 1. 概述

### 1.1 产品定位

Vibe Design 是 AI 驱动的产品设计助手，帮助用户从想法到完整的产品需求文档（PRD），包括用户画像、竞品分析和技术可行性评估。

**核心理念**: "描述你的想法，AI 帮你完善产品"

### 1.2 核心价值

| 为谁 | 解决什么问题 | 达成什么结果 |
|------|-------------|-------------|
| 独立开发者 | 有想法但缺乏产品设计能力 | 快速生成结构化 PRD |
| 设计师/产品经理 | 需要快速输出产品文档 | AI 辅助提升效率 10x |
| 技术创业者 | 不懂产品验证方法 | AI 提供竞品分析和差异化建议 |
| 内容创作者 | 想法多但难以落地 | 结构化引导完成产品定义 |

### 1.3 关键指标

| 指标 | 定义 | 目标值 | 当前值 |
|------|------|--------|--------|
| PRD 生成时间 | 从输入到 AI 输出 PRD 的时间 | < 30s | ✅ 达标 |
| PRD 采纳率 | 用户采纳 AI 建议的比例 | > 80% | - |
| 用户满意度 | 用户对生成结果的评分 | > 4.5/5 | - |
| 平均交互轮次 | 完成一次设计会话的交互次数 | < 5 轮 | - |

## 4. AI 提示词模板

### 4.1 PRD 生成提示词

```markdown
你是一位经验丰富的产品经理，擅长将模糊的想法转化为结构化的产品需求文档。

## 用户想法
{user_idea}

## 任务
请根据用户的想法，生成一份完整的产品需求文档（PRD），包含以下内容：

1. **产品名称**: 起一个简洁、易记、能体现产品价值的名字
2. **一句话简介**: 用一句话清晰描述产品定位
3. **目标用户**: 明确 2-3 个主要用户群体
4. **核心功能**: 列出 3-5 个最核心的功能，按优先级排序
5. **技术栈建议**: 推荐适合的技术栈（前端/后端/数据库）
6. **开发工作量**: 预估开发时间（人天）和难度等级（低/中/高）
7. **商业模式**: 简要说明可能的盈利方式

## 输出格式
请使用 Markdown 格式，结构清晰，便于阅读。
```

### 4.2 用户画像提示词

```markdown
你是一位用户研究专家，擅长创建生动的用户画像。

## 产品信息
{product_info}

## 任务
请为目标用户群体创建 2-3 个详细的用户画像，每个画像包含：

1. **基本信息**: 姓名、年龄、职业、收入、地理位置
2. **核心需求**: 他们最迫切需要解决的问题
3. **行为特征**: 日常习惯、工具使用偏好
4. **痛点描述**: 当前解决方案的不满之处
5. **使用场景**: 在什么情况下会使用这个产品
6. **用户故事**: 以"As a..., I want to..., so that..."格式编写

## 输出格式
每个画像使用卡片形式，包含 emoji 图标增强可读性。
```

### 4.3 竞品分析提示词

```markdown
你是一位市场分析师，擅长竞品调研和差异化策略。

## 产品信息
{product_info}

## 任务
请进行竞品分析，包含：

1. **竞品识别**: 列出 3-5 个直接或间接竞品
2. **功能对比**: 对比各竞品的核心功能和特点
3. **定价策略**: 分析竞品的定价模式
4. **优劣势分析**: 每个竞品的优势和劣势
5. **市场机会**: 找出市场空白和差异化机会
6. **建议策略**: 给出差异化和竞争策略建议

## 输出格式
使用表格对比，清晰直观。
```

## 6. 数据结构

### 6.1 PRD 数据模型

```typescript
interface ProductRequirement {
  id: string;
  name: string;
  tagline: string;
  description: string;
  targetUsers: TargetUser[];
  coreFeatures: Feature[];
  techStack: TechStack;
  estimatedEffort: EffortEstimate;
  businessModel?: string;
  createdAt: number;
  updatedAt: number;
}

interface TargetUser {
  name: string;
  description: string;
  needs: string[];
  painPoints: string[];
}

interface Feature {
  title: string;
  description: string;
  priority: 'P0' | 'P1' | 'P2';
  userStories: string[];
}

interface TechStack {
  frontend: string[];
  backend: string[];
  database: string[];
  reasoning: string;
}

interface EffortEstimate {
  days: number;
  difficulty: 'low' | 'medium' | 'high';
  assumptions: string[];
}
```

### 6.2 用户画像数据

```typescript
interface UserPersona {
  id: string;
  name: string;
  age: number;
  occupation: string;
  income?: string;
  location?: string;
  avatar?: string; // emoji
  bio: string;
  needs: string[];
  painPoints: string[];
  behaviors: string[];
  scenarios: string[];
  userStories: UserStory[];
}

interface UserStory {
  asA: string;
  iWant: string;
  soThat: string;
}
```

### 6.3 竞品分析数据

```typescript
interface CompetitorAnalysis {
  competitors: Competitor[];
  comparisonMatrix: ComparisonMatrix;
  swotAnalysis: SWOT;
  recommendations: Recommendation[];
}

interface Competitor {
  name: string;
  url?: string;
  category: 'direct' | 'indirect';
  features: string[];
  pricing?: string;
  strengths: string[];
  weaknesses: string[];
}

interface ComparisonMatrix {
  dimensions: string[];
  products: { name: string; scores: number[] }[];
}

interface SWOT {
  strengths: string[];
  weaknesses: string[];
  opportunities: string[];
  threats: string[];
}

interface Recommendation {
  type: 'differentiation' | 'pricing' | 'feature' | 'marketing';
  description: string;
  priority: 'high' | 'medium' | 'low';
}
```
