## 📖 文档规范

**详细的 Sprint 文档编写规范请查看**: [sprint-guide.md](./sprint-guide.md)

### 快速参考

#### 文档命名
`sprint-{N}.md`  
示例：`sprint-1.md`, `sprint-2.md`

#### 优先级定义
- **P0**: Must Have（必须完成）
- **P1**: Should Have（应该完成）
- **P2**: Nice to Have（锦上添花）

#### 状态标识
- ✅ 已完成
- 🔄 进行中
- 📋 待开始
- ❌ 已取消

#### 敏捷层次
```
Epic → Feature → User Story → Technical Task
```

## 🔄 Sprint 工作流程

### 1. Sprint 规划
**时间**: Sprint 开始前 1-2 天  
**产出**: Sprint 规划文档

### 2. Sprint 执行
**日常**: 每日站会（15 分钟）+ 任务开发

### 3. Sprint 评审
**时间**: Sprint 最后一天  
**参与**: 团队 + 干系人  
**内容**: Demo 展示 + 反馈收集

### 4. Sprint 回顾
**时间**: Sprint 结束后 1-2 天  
**产出**: 改进行动计划

---

## 📈 关键指标

### 进度指标
- **计划完成率**: 实际完成 / 计划任务 × 100%
- **速度**: 每个 Sprint 完成的故事点数

### 质量指标
- **Bug 率**: Bug 数 / 完成任务数 × 100%
- **测试覆盖率**: ≥80%
